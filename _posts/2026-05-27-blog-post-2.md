# Pupil Dilation Analysis Pipeline

In this blog post, I present a complete pipeline for analyzing pupil dilation data. The goal of this pipeline is to identify when pupil dilation differs between experimental conditions. It spans the full workflow from extracting pupil diameter from raw eye-tracking files to performing cluster-based permutation analysis.

The example dataset is drawn from my ongoing eye-tracking study, which is currently under peer review.

Please note that the experimental design is not intended to directly compare pupil dilation between conditions. Instead, the study focuses on differences in looking proportions across conditions. Here, the dataset is used purely to demonstrate how pupil diameter can be extracted, preprocessed, and analyzed. This pipeline will be updated once a more suitable example dataset is available.

---

## Pipeline Overview

The analysis pipeline consists of four main steps:

---

## Step 1: Extracting Pupil Diameter from EDF Files  
**Script:** `step1_Get_diameter_from_EDF.m`

1. This script loads raw EDF eye-tracking files and extracts event markers along with continuous pupil size data. It aligns these signals into trial-based structures using timestamp information and trial start/end events.

2. Each sample is then annotated with trial-level metadata, including stimulus information, condition labels, and behavioral accuracy. This is achieved by matching eye-tracking data with an accompanying experimental design spreadsheet (Excel file).

3. Finally, the script outputs a cleaned trial-wise sample table for each participant, which is saved for downstream preprocessing and analysis.

---

## Step 2: Preprocessing and Epoching  
**Script:** `step2_Preprocessed.m`

1. The cleaned eye-tracking data are segmented into trial-based epochs aligned to stimulus onset.

2. Each pupil signal undergoes preprocessing, including:
   - Removal of blinks (NaNs and extreme outliers)
   - Interpolation of short missing segments
   - Smoothing of the time series
   - Baseline correction using a pre-stimulus interval

3. Trials with poor data quality are excluded based on predefined criteria (e.g., excessive missing data, flat signals, insufficient valid samples). Trials with no response or incorrect responses are also removed. The resulting preprocessed trial-level pupil data are saved individually for each subject.

---

## Step 3: Preparing Data for Cluster-Based Permutation Analysis  
**Script:** `step3_prepare_for_cluster.m`

1. This step loads all preprocessed single-trial pupil epochs and removes previously rejected trials.

2. The data are reshaped into a long-format table containing:
   - Time
   - Pupil size
   - Subject ID
   - Trial ID
   - Experimental condition

   Subject-level averages are computed and interpolated onto a common time window (0–780 ms).

3. Finally, grand-average pupil responses are computed for each condition, including variability estimates (standard error of the mean). The script generates condition-wise time series plots with shaded error bands and saves both the aggregated dataset and figures for further statistical analysis.

---

## Step 4: Cluster-Based Permutation Analysis  
**Script:** `step4_cluster.R`

1. A time-resolved (bin-wise) repeated-measures ANOVA is first conducted across all time points to test for condition effects in pupil dilation.

2. Temporal clusters are then identified by grouping consecutive time points where the effect exceeds a predefined statistical threshold.

3. A null distribution is generated using 1,000 within-subject permutations, where condition labels are shuffled within participants. For each permutation, the same clustering procedure is applied to estimate cluster statistics under the null hypothesis.

4. Finally, corrected cluster-level p-values are computed by comparing observed clusters against the null distribution. Significant clusters are saved for reporting, and cluster boundary thresholds are extracted for later visualization and interpretation.
